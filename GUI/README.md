# YOLO目标检测系统 - Gradio版本

基于Gradio构建的YOLO目标检测Web界面，是原PySide6(Qt)版本的重构实现。

## 🎯 功能特性

- 📷 **单张图片检测** - 上传图片进行目标检测
- 🎬 **视频检测** - 处理视频文件并生成检测结果
- 📹 **实时摄像头检测** - 实时流式目标检测
- 📂 **批量检测** - 对整个文件夹进行批量处理
- 🎚️ **置信度调整** - 动态调整检测置信度阈值
- 📊 **详细统计** - 检测目标统计和类别分析

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 准备YOLO模型

将 `.pt` 格式的YOLO模型文件放在以下任一目录：
- `models/`
- `pt_models/`
- `weights/`
- `yolo_models/`
- 用户主目录下的 `yolo_models/`

### 3. 启动应用

```bash
python run.py
```

或使用命令行直接启动：

```bash
python gradio_app.py
```

### 4. 访问界面

打开浏览器访问：http://localhost:7860

## 🛠️ 技术栈

- **Gradio** - Web界面框架
- **Ultralytics YOLO** - 目标检测模型
- **OpenCV** - 图像/视频处理
- **NumPy** - 数值计算
- **Pillow** - 图像处理

## 📝 使用说明

### 图片检测
1. 选择"图片检测"标签页
2. 上传待检测的图片
3. 选择模型并加载
4. 调整置信度阈值（可选）
5. 点击"开始检测"

### 视频检测
1. 选择"视频检测"标签页
2. 上传视频文件
3. 点击"开始处理"
4. 等待处理完成后查看结果

### 实时检测
1. 选择"实时检测"标签页
2. 允许浏览器访问摄像头
3. 系统自动开始实时检测

### 批量检测
1. 选择"批量检测"标签页
2. 输入包含图片的文件夹路径
3. 点击"开始批量检测"
4. 查看汇总报告和保存路径

## ⚙️ 配置选项

### 置信度阈值
- 范围：0.01 - 1.0
- 默认值：0.25
- 说明：越高则检测结果越严格

### 模型搜索路径
系统自动在以下路径搜索 `.pt` 模型文件：
- 当前目录下的 `models/`, `pt_models/`, `weights/`, `yolo_models/`
- 用户主目录下的 `yolo_models/`
- Windows: `C:/yolo_models/`
- Linux: `/usr/local/share/yolo_models/`

## 🔧 高级功能

### 自定义模型路径
修改 `model_manager.py` 中的 `models_paths` 列表添加自定义路径。

### 修改界面样式
编辑 `gradio_app.py` 中的 `custom_css` 变量自定义样式。

### 调整服务参数
在 `run.py` 或 `gradio_app.py` 中修改 `demo.launch()` 的参数：
- `server_port` - 服务端口
- `share` - 是否创建公开链接
- `show_error` - 是否显示详细错误信息

## 🐛 常见问题

### 1. 摄像头无法访问
- 确保浏览器允许摄像头权限
- 检查摄像头是否被其他程序占用
- 尝试刷新页面重新授权

### 2. 模型加载失败
- 确认模型文件格式为 `.pt`
- 检查模型文件是否完整
- 查看控制台错误信息

### 3. 视频处理卡顿
- 降低视频分辨率
- 减少处理的帧数
- 升级硬件配置

### 4. 内存不足
- 批量检测时减少同时处理的图片数量
- 关闭不必要的程序释放内存
- 使用更低分辨率的输入
